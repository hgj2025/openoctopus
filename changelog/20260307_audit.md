# 审计模块完善（2026-03-07）

## 背景

本次改动完善了 OpenOctopus 的安全审计体系，覆盖消息记录、Skill 安装审计与安全拦截、系统命令白名单三个核心模块。

## 改动详情

### 1. 消息记录（无需代码修改）

`extensions/audit` 插件已通过 `message_received` 钩子实现 `user.message` 事件的结构化记录，写入 `~/.openclaw/audit/audit.jsonl`。

**启用方式**：在配置中设置 `audit.enabled = true` 即可。

---

### 2. Skill 安装审计与安全门控

**新增功能**：
- 新增 `skill.install` 审计事件类型，记录每次 Skill 安装的意图（`phase: "before"`）和结果（`phase: "after"`），包含安全扫描警告、是否存在高危模式（`hasCritical`）、是否被中止（`blocked`）等字段。
- `SkillInstallRequest` 新增 `onDangerousConfirm` 回调接口：当扫描发现高危代码模式时，调用该回调由上层决定是否继续安装；若回调返回 `false`，则安装被中止并记录 `blocked: true`。

**涉及文件**：
- `src/agents/skills-install.ts` — 新增确认回调与中止逻辑
- `src/plugins/types.ts` — 新增 `skill_install` 钩子类型及 `PluginHookSkillInstallEvent`
- `src/plugins/hooks.ts` — 新增 `runSkillInstall()` 方法
- `src/plugin-sdk/index.ts` — 导出 `PluginHookSkillInstallEvent` 及相关钩子类型
- `src/gateway/server-methods/skills.ts` — 在安装前后触发 `skill_install` 钩子
- `extensions/audit/src/types.ts` — 新增 `SkillInstallEvent` 类型
- `extensions/audit/src/event-builder.ts` — 新增 `buildSkillInstallEvent()`
- `extensions/audit/src/config-schema.ts` — `eventFilter` 枚举新增 `skill.install`；新增 `commandPolicy` 配置块
- `extensions/audit/src/index.ts` — 订阅 `skill_install` 钩子并写入审计日志；将 `commandPolicy` 传入 `InterceptionEngine`

---

### 3. 系统命令安全审计 / 白名单

**新增功能**：
- 审计插件配置新增 `commandPolicy` 块，支持三种模式：
  - `whitelist`：仅允许 `allowedCommands` 列表中的命令
  - `blacklist`：拒绝 `blockedCommands` 列表中的命令
  - `audit_only`：仅记录，不拦截
- 新增 `extractBaseCommand()` 工具函数，可正确处理 `sudo`、环境变量前缀（`KEY=val cmd`）、绝对路径（`/usr/bin/git`）等场景，提取真实的基础命令名。
- 命令策略检查在所有自定义拦截规则之前执行，被拦截的命令自动写入 `tool.blocked` 审计事件。

**涉及文件**：
- `extensions/audit/src/config-schema.ts` — 新增 `commandPolicy` Zod schema
- `extensions/audit/src/interception.ts` — 新增 `extractBaseCommand()` 及命令策略评估逻辑
- `extensions/audit/src/index.ts` — 将 `commandPolicy` 传入 `InterceptionEngine` 构造函数

---

## 验证步骤

1. **消息记录**：配置 `audit.enabled = true`；发送 Feishu 消息；检查 `~/.openclaw/audit/audit.jsonl` 中的 `user.message` 事件。

2. **Skill 安装审计**：
   - 通过 gateway 安装任意 Skill；检查审计日志中是否出现 `phase: "before"` 和 `phase: "after"` 的 `skill.install` 事件。
   - 安装含高危代码扫描结果的 Skill（模拟或真实）；验证 `hasCritical: true` 是否被记录；若 `onDangerousConfirm` 返回 `false`，验证 `blocked: true`。

3. **命令白名单**：
   - 启用 `audit.commandPolicy.enabled = true`，`mode: "whitelist"`，`allowedCommands: ["git", "node"]`
   - 启用 `audit.interception.enabled = true`
   - 让 AI 执行 `git status` → 应成功
   - 让 AI 执行 `curl http://example.com` → 应被拦截；检查审计日志中 `tool.blocked` 事件，`ruleId` 为 `"command-policy-whitelist"`
   - 执行 `sudo rm -rf /tmp/x` → 应被拦截（基础命令 `rm` 不在白名单中）

4. 运行 `pnpm test` 和 `pnpm tsgo` 确认无类型错误或回归。
