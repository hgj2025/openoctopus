梳理一下完整的进程树：

  启动后稳定运行时：

  bash dev.sh
  └── node scripts/dev-all.mjs          # 我们的管理进程
      ├── node scripts/run-node.mjs      # 后端 runner（监控 openclaw）
      │   └── node openclaw.mjs          # 真正的 gateway 进程
      └── node scripts/ui.js dev         # 前端 runner
          └── pnpm run dev               # pnpm 脚本
              └── vite                   # Vite dev server

  共 6 个进程（稳定态）。另外有 2 个临时进程在启动时出现随后退出：

  - pnpm tsdown exec tsdown --no-clean — 后端首次构建（build 完就退）
  - pnpm install — 前端 deps 检查（已安装则跳过）

  所以实际上 run-node.mjs 是一个 wrapper，它负责先 build 再启动真正的 openclaw.mjs，并在 openclaw 退出时重新
  build+restart。
